# Chess Analysis App — Дизайн-прототип

## Джерело

Дизайн-прототип лежить у папці:

```text
design-prototype/
```

Основний файл для перегляду:

```text
design-prototype/Chess Analysis App.html
```

Це автономний HTML/React-прототип, згенерований через Claude design workflow. Він використовується як візуальний і UX-референс, а не як production-код.

## Файли прототипу

```text
design-prototype/
  Chess Analysis App.html
  chess-board.jsx
  chess-pieces.jsx
  design-canvas.jsx
  tweaks-panel.jsx
```

Призначення:

- `Chess Analysis App.html` — основний перегляд прототипу.
- `chess-board.jsx` — кастомна шахова дошка для прототипу.
- `chess-pieces.jsx` — SVG-фігури для прототипу.
- `design-canvas.jsx` — canvas/wrapper для design workflow.
- `tweaks-panel.jsx` — панель налаштувань прототипу.

## Статус

- Прототип прийнятий як UI-референс.
- Прототип не є джерелом правди для архітектури.
- Прототип не є production implementation.
- Production має реалізовуватись на Next.js 15 + CSS Modules згідно зі спекою.

## Що беремо з прототипу

### Загальний настрій

- Темна шахова workspace-естетика.
- Зелена/лісова палітра.
- Спокійний, сфокусований інтерфейс.
- Відчуття інструмента для аналізу, а не маркетингової сторінки.

### Навігація

- Desktop: вузький sidebar з іконками.
- Mobile: bottom navigation.
- Основні розділи:
  - дашборд;
  - аналіз;
  - бібліотека/список партій;
  - профіль.

### Дашборд

- Верхній блок із коротким привітанням і контекстом.
- Компактні stat cards.
- Помітний блок імпорту / підключення платформи.
- Список останніх партій як компактні рядки.

Для production треба адаптувати блок імпорту під поточну спеку:

- onboarding/settings керують підключенням Chess.com / Lichess акаунтів;
- dashboard показує фоновий статус синхронізації;
- нові партії підтягуються автоматично при вході або оновленні dashboard.

### Сторінка аналізу партії

- Дошка як центральний об'єкт.
- Eval bar поруч із дошкою.
- Move navigation під дошкою.
- Права панель із:
  - гравцями;
  - accuracy;
  - помилками;
  - списком ходів;
  - eval graph;
  - коментарями.

Цей layout добре відповідає цілі Stockfish game review.

### Профіль

- Візуальний блок користувача.
- Платформи як бейджі.
- Графік динаміки рейтингу.
- Статистика дебютів.

Для production профіль використовує фільтр періоду: 7 / 30 / 90 днів або всі партії.

## Що адаптуємо обережно

- Прототип використовує inline styles; у production переносити в CSS Modules.
- Прототип використовує CDN React + Babel; у production цього не має бути.
- Прототип має кастомну дошку і SVG-фігури; production за спекою використовує `react-chessboard`.
- Прототип має tweak panel; production не потребує цієї панелі.
- Прототип містить sample data; production має працювати з БД і API.
- Прототип показує "Вставити PGN"; у v1 це не зафіксовано як основний імпорт, тому не переносити без окремого рішення.

## Типографіка

Прототип використовує:

- `DM Sans` для основного UI;
- `DM Mono` для чисел, evaluation, rating і compact stats.

У production можна використати ці шрифти, якщо це не конфліктує з performance і Next.js font setup.

## Відхилення від спеки

У прототипі є елементи, які не є частиною погодженого v1:

- вставка PGN як основний шлях імпорту;
- бібліотека відомих історичних партій;
- кастомний board engine замість `react-chessboard`;
- tweak panel;
- статичні rating/sample metrics.

Ці речі не переносити в production без окремого рішення.

## Правило використання

Під час реалізації UI:

1. Відкрити `design-prototype/Chess Analysis App.html` як візуальний референс.
2. Звірити поведінку зі `spec/ui-context.md`.
3. Реалізувати в Next.js + CSS Modules, не копіюючи прототип напряму.
4. Якщо потрібно відійти від прототипу, пріоритет має специфікація.
