# Chess Analysis App — Базова специфікація

## Огляд

Вебзастосунок для аналізу шахових партій з Chess.com та Lichess із використанням LLM-аналітики. Користувач вводить нікнейм, завантажує свої партії та отримує тактичний і стратегічний аналіз — як по окремій партії, так і по кількох за певний період.

Проєкт реалізується маленькими фазами. Кожна фаза має завершуватись робочим станом продукту, а не частково зібраним великим релізом.

**Базове рішення v1:** спочатку авторизація, потім весь функціонал. Імпорт, збереження партій, Stockfish-аналіз, LLM-аналіз і профіль доступні лише авторизованому користувачу.

## Основні функції

### 1. Автентифікація
- Вхід через GitHub OAuth
- Сесійна авторизація через NextAuth.js v5
- Session strategy: JWT; таблиця `auth_sessions` у v1 не потрібна
- Весь основний функціонал вимагає авторизації
- Анонімний імпорт / анонімний аналіз не входить у v1
- Email + пароль можна додати пізніше, якщо буде реальна потреба

### 2. Імпорт партій
- Введення: платформа (Chess.com / Lichess) + нікнейм
- У v1 підтримуються обидві платформи одразу: Chess.com і Lichess
- Імпорт не тягне всю історію за весь час
- Користувач вибирає один із двох режимів імпорту:
  - за кількістю: 25 / 50 / 100 найновіших партій (загальна кількість, об'єднана по всіх контролях часу)
  - за періодом: партії за останні 7 / 30 / 90 днів (об'єднані по всіх контролях часу)
- Значення за замовчуванням: режим за кількістю, 25 партій
- Завантажуються партії всіх основних контролів часу в межах вибраного режиму: класика, рапід, бліц, пуля. Парам збираються по всіх контролях часу, об'єднуються за датою, сортуються від новіших, і результат обрізається до запитаної кількості або часового періоду. Наприклад, для режиму "25 партій" система зберігає останні 25 партій (незалежно від контролю часу), запускаючи імпорт по черзі для класики, рапіду, бліцу і пулі, сортуючи результати за датою.
- Chess.com API: `https://api.chess.com/pub/player/{user}/games/{year}/{month}`
- Lichess API: `https://lichess.org/api/games/user/{user}` (NDJSON-стрим)
- Партії зберігаються в БД прив'язаними до облікового запису після першого завантаження
- Повторне завантаження оновлює лише нові партії

### 3. Список і фільтрація партій
- Пагінований список імпортованих партій
- Фільтри: платформа, контроль часу, результат (перемога/поразка/нічия)
- По кожній партії: суперник, результат, дебют, дата, статус аналізу

### 4. Інтерактивна дошка (перегляд однієї партії)
- Відтворення партії хід за ходом (назад / вперед / початок / кінець)
- Список ходів (PGN-нотація)
- Підсвічування останнього ходу на дошці
- Дефолтна Stockfish-аналітика у стилі WintrChess / Chess.com game review:
  - оцінна шкала позиції (eval bar)
  - графік оцінки партії по ходах
  - стрілка або підсвічування найкращого ходу
  - ключові моменти / turning points
  - accuracy для гравця і суперника
  - класифікація ходів: brilliant / best / good / inaccuracy / mistake / blunder
- Класифікація ходів базується на стандартній системі втрати centipawn / win probability loss; користувацького налаштування порогів у v1 немає
- Базовий профіль рушія: Stockfish WASM у браузері з фіксованими дефолтними налаштуваннями; точні `depth` / час на позицію можна зафіксувати після короткого performance spike
- Результати engine-аналізу зберігаються в БД для повторного перегляду, статусу аналізу в списку партій і використання в LLM-зведеннях
- Зберігається лише результат дефолтного профілю аналізу, не всі проміжні варіанти / глибини

### 5. LLM-аналіз однієї партії
- Запускається вручну ("Аналізувати цю партію")
- LLM отримує: повний PGN, колір гравця, результат, контроль часу, ключові моменти Stockfish-аналізу (якщо вже є)
- Мова відповіді у v1: українська
- Формат відповіді: строгий JSON за узгодженою схемою, не довільний markdown
- Структура виводу LLM:
  - **Загальна оцінка** (2-3 речення)
  - **Дебют** — чи зіграно правильно, що пішло не так
  - **Мідлгейм** — ключові моменти, пропущена тактика, позиційні помилки
  - **Ендшпіль** — оцінка техніки, якщо дійшло до нього
  - **Критичні моменти** — до 3 конкретних позицій (номер ходу + опис)
  - **Рекомендації** — 2-3 конкретні речі для роботи над собою
- Аналіз кешується в БД — повторний аналіз коштує API-виклику, пропонується як опція

### 6. Груповий аналіз (LLM-аналіз кількох партій)
- Користувач вибирає 5–30 партій зі свого списку (вибір через чекбокс)
- LLM отримує: стислі зведення по вибраних партіях (не повний PGN — для вміщення у контекстне вікно)
- Зведення партії = результат, дебют, контроль часу, кількість ходів, engine-класифікації / accuracy / ключові моменти (якщо є), нотатки попереднього LLM-аналізу (якщо є)
- Мова відповіді у v1: українська
- Формат відповіді: строгий JSON
- Вивід LLM:
  - **Закономірності у вибраних партіях** — повторювані помилки
  - **Тактичні слабкості** — конкретні типи патернів (зв'язки, виделки, мат на останній горизонталі тощо)
  - **Стратегічні слабкості** — структура пішаків, активність фігур, управління часом
  - **Оцінка дебютів** — які дебюти шкодять гравцеві
  - **Пріоритизований план дій** — топ-3 речі для покращення

### 7. Профіль гравця та накопичувальне зведення
- Відображається на сторінці профілю після аналізу достатньої кількості партій (мінімум 5)
- **Дашборд метрик**:
  - вибір кількості партій: 25 / 50 / 100
  - вибір періоду: 7 / 30 / 90 днів
  - значення за замовчуванням: 25 партій і 7 днів
  - Розбивка перемог / нічиїх / поразок (кругова або стовпчаста діаграма)
  - Результативність за контролем часу
  - Результативність за кольором (білі vs чорні)
  - Найбільш зіграні дебюти + відсоток перемог по кожному дебюту
- **LLM-наративне зведення** (оновлюється за запитом):
  - Письмова оцінка систематичних помилок гравця
  - Що покращується, а що стоїть на місці
  - Налаштовано на ~1000 ELO — практично, конкретно, без термінів рівня гросмейстера

## Стратегія LLM

### Вибір моделі
- **Основна:** Gemini Flash 2.0 (`gemini-2.0-flash`) через Google AI Studio — безкоштовний тариф: 15 запитів/хв, 1M токенів/день
- **Запасна:** Groq (`llama-3.3-70b-versatile`) — безкоштовний тариф: 14 400 запитів/день

### Управління контекстом
- Одна партія: надсилається повний PGN (~500-2000 токенів) + системний промпт
- Груповий аналіз: надсилаються стиснені зведення, щоб вміститись у 8k токенів
- Промпти завжди включають ELO-контекст гравця для калібрування порад
- Всі відповіді LLM у v1 генеруються українською
- Всі LLM-відповіді мають бути валідним JSON, який можна зберегти і відобразити в UI без парсингу markdown

### Структура промпту (системна)
```
You are a chess coach analyzing games for a ~1000 ELO player.
Focus on practical, actionable advice. Avoid grandmaster-level concepts.
Point out concrete mistakes with move numbers. Be direct and constructive.
Respond in Ukrainian.
Return strict JSON matching the requested schema.
```

## Відкриті питання / Ризики

| Питання | Статус |
|---|---|
| Ліміти частоти запитів Chess.com API навіть при імпорті 25/50/100 партій | Перевірити |
| NDJSON-стримінг Lichess у Next.js API route — потрібна уважна обробка | Перевірити |
| Чи вистачить безкоштовного тарифу Gemini для реального використання? Потрібен запасний варіант Groq? | Моніторити після запуску |

| Точні дефолтні пороги класифікації ходів | Взяти стандартну centipawn / win probability loss модель, без кастомізації у v1 |
