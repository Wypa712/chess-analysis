  <action>
    Read ALL module CSS files listed above. For each file, perform the following audit checks and apply fixes:

    **Colour token audit:**
    - Find any hardcoded hex values (e.g., `#d8ead8`, `#8aa98a`, `#2d433a`) that have an equivalent CSS custom property in `globals.css` (e.g., `var(--color-text)`, `var(--color-text-muted)`, `var(--color-border)`).
    - Replace hardcoded hex values with the corresponding CSS variable. Do NOT replace values that are unique to the component and have no matching global token — leave those as-is.
    - Exception: `rgba(...)` with inline opacity adjustments are acceptable.

    **Typography audit:**
    - Ensure `font-family` declarations use `var(--font-sans)` or `var(--font-mono)` (not hardcoded stack strings that duplicate globals).
    - Ensure heading sizes are consistent across pages: h1-equivalent elements should use similar font-size values.
    - For numeric values (ratings, counts, ELO numbers) that use monospace: confirm `font-family: var(--font-mono)` is applied.

    **Spacing audit:**
    - Identify inconsistent gap/padding values. Common values in the codebase: 4px, 8px, 12px, 16px, 24px, 32px. Flag and fix any component that uses unusual one-off values (e.g., `padding: 7px 11px`) that break the 4px grid.
    - Ensure card/panel components have consistent `border-radius` using `var(--radius-sm)`, `var(--radius-md)`, or `var(--radius-lg)`.

    **Loading state audit (D-17):**
    - Read component files (`.tsx`) for the pages to understand what loading state is used where:
      - Dashboard: check if `GamesList` uses skeleton or spinner.
      - Game page: check `analysisState === "loading"` path — uses spinner or skeleton?
      - Profile page: check `initialLoading` path.
      - Onboarding: check progress state.
    - Apply consistency rule: full-page initial loads should use spinner (full-page overlay or large centered indicator); individual data sections within an already-loaded page should use skeleton.
    - If a component uses a spinner where a skeleton fits better (or vice versa), adjust the CSS to match the correct pattern. Do NOT add new spinner components — only restyle or adjust what exists.

    Apply all fixes directly to the CSS files. Document fixes as inline CSS comments: `/* Audit: replaced #d8ead8 → var(--color-text) */`.
  </action>
