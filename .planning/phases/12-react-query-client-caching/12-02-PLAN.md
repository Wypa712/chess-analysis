## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→/api/games | Запити до API авторизовані через next-auth session cookie |
| userId в queryKey | userId береться з useSession() — контрольований auth шаром |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-02-01 | Information Disclosure | queryKey з userId | accept | userId вже є в браузері через session; queryKey не передається на сервер; кеш локальний до браузера |
| T-12-02-02 | Tampering | cache invalidation | accept | invalidateQueries лише примушує рефетч з API; API сам перевіряє auth і повертає тільки дані поточного юзера |
| T-12-02-03 | Spoofing | useSession userId | mitigate | userId з useSession контролюється next-auth; серверний API /api/games незалежно верифікує сесію — подвійна перевірка |
</threat_model>
